# Code Quality Rule

## Cognitive Complexity チェック

コードの認知的複雑度を [thoughtbot/complexity](https://github.com/thoughtbot/complexity) で計測する。

### 実行方法

```bash
nix run nixpkgs#cargo -- install complexity
complexity <対象ディレクトリ>
```

### 使い方

- `complexity .` で現在のディレクトリ以下を計測
- `--format json` または `--format csv` で出力形式を変更
- `--only .py,.rs` で対象ファイル拡張子を絞り込み
- `.gitignore` に記載されたパスは自動的に除外される

### 判断基準

スコアが高いファイルほど認知的複雑度が高い。スコアが突出しているファイルはリファクタリング候補として報告する。

## コミット前の品質確認

コミットを作成する前に、以下の品質指標を確認する。いずれかの基準を満たさない場合、修正してからコミットすること。

### 確認項目

| 指標 | 確認方法 | 基準 |
|------|---------|------|
| Cognitive Complexity | `complexity <対象ディレクトリ>` | 変更したファイルのスコアが変更前より悪化していないこと |
| テストカバレッジ | プロジェクトのテストランナーで計測 | 変更したファイルのカバレッジが低下していないこと |

### フロー

1. 実装完了後、`complexity` で変更対象ファイルの複雑度を計測する
2. テストカバレッジを計測する
3. 両指標が基準を満たしていればコミットを作成する
4. 基準を満たさない場合はリファクタリングまたはテスト追加を行い、再計測する
