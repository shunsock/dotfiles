## 概要

変更ファイルに対して認知的複雑度を定量計測し、合否を判定する。
循環的複雑度・凝集度・結合度は定性的な所見として提供する（合否判定対象外）。

## Step 1: 認知的複雑度の計測

[thoughtbot/complexity](https://github.com/thoughtbot/complexity) CLI を使用する。
このツールはインデントベースのヒューリスティックで、コードの読みにくさを数値化する。

```bash
complexity --format json {対象ファイル or ディレクトリ}
```

### 合否基準

- **合格**: スコア **15 以下**
- **不合格**: スコア **16 以上**

SonarSource が認知的複雑度の推奨上限として 15 を提唱しており、これに準拠する。

### complexity CLI が利用できない場合

`which complexity` で存在を確認し、コマンドが見つからない場合はユーザーに報告する。
この場合、認知的複雑度の判定は「判定不能」とし、不合格にはしない。

## Step 2: 定性評価（参考所見）

以下の観点はコードを読んで定性的に評価する。
言語を問わず適用できるが、合否判定には含めない。気になる点があれば参考所見として報告する。

### 循環的複雑度（Cyclomatic Complexity）

- 関数・メソッド内の分岐（if / switch / ループ）が過度に多くないか
- 実行パスの数が多すぎてテストが困難になっていないか

### 凝集度（Cohesion）

- モジュール内のクラス・関数が単一の責務に集中しているか
- 無関係な機能が1つのモジュールに混在していないか

### 結合度（Coupling）

- 他モジュールへの import が過度に多くないか
- 具体的な実装ではなく抽象（インターフェース・プロトコル）に依存しているか

## 出力形式

以下の形式で結果を返すこと:

```markdown
### 判定
合格 or 不合格

### 認知的複雑度（complexity CLI）
| ファイル | スコア | 判定 |
|---------|--------|------|
| path/to/file.py | 12.3 | 合格 |

### 定性評価（参考所見）
（循環的複雑度・凝集度・結合度について。合否判定対象外。気になる点があれば記載）

### 指摘事項（不合格の場合）
- ファイル名: 指摘内容と改善案
```
